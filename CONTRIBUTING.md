# 🤝 Contributing to API Shield

First off, thank you for considering contributing to API Shield! ❤️

We appreciate every contribution, whether it's fixing a bug, improving documentation, adding a new feature, or reporting an issue.

Our goal is to build a secure, reliable, and developer-friendly middleware toolkit for the Node.js ecosystem.

---

# Getting Started

## 1. Fork the Repository

Fork this repository to your GitHub account.

## 2. Clone Your Fork

```bash
git clone https://github.com/<your-username>/api-shield.git
```

## 3. Install Dependencies

```bash
npm install
```

## 4. Start Development

```bash
npm run dev
```

---

# Development Workflow

1. Create a new branch.

```bash
git checkout -b feature/your-feature-name
```

2. Make your changes.

3. Run tests.

```bash
npm test
```

4. Build the project.

```bash
npm run build
```

5. Commit your changes.

```bash
git commit -m "feat: add request logger"
```

6. Push your branch.

```bash
git push origin feature/your-feature-name
```

7. Open a Pull Request.

---

# Coding Standards

Please follow these guidelines:

- Write clean and readable code.
- Follow the existing project structure.
- Keep functions small and focused.
- Add comments only when necessary.
- Prefer TypeScript over JavaScript.
- Avoid breaking existing APIs.

---

# Commit Message Convention

We follow the Conventional Commits specification.

Examples:

```text
feat: add rate limiter

fix: resolve ip blocker issue

docs: update README

refactor: simplify middleware loader

test: add unit tests
```

---

# Pull Request Guidelines

Before submitting a Pull Request, please ensure:

- Project builds successfully.
- All tests pass.
- Code is properly formatted.
- Documentation is updated if required.
- Your changes do not introduce breaking changes.

---

# Reporting Bugs

When reporting a bug, please include:

- Node.js version
- Package version
- Operating System
- Steps to reproduce
- Expected behavior
- Actual behavior

---

# Feature Requests

Feature requests are always welcome.

Please open an issue describing:

- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered
