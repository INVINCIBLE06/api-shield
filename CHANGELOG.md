## Versioning Strategy

### MAJOR

Breaking API changes.

Example:

```
1.0.0 → 2.0.0
```

### MINOR

New features without breaking existing APIs.

Example:

```
1.1.0 → 1.2.0
```

### PATCH

Bug fixes and performance improvements.

Example:

```
1.2.0 → 1.2.1
```
