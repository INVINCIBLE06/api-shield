# 🔒 Security Policy

Thank you for helping keep **API Shield** secure.

The security of our users is our highest priority. We appreciate responsible disclosure of security vulnerabilities and are committed to addressing them promptly.

---

# Supported Versions

The following versions of API Shield are currently supported with security updates.

| Version | Supported |
|---------|-----------|
| 1.x.x | ✅ Yes |
| 0.x.x | ❌ No |

> Only the latest stable release receives security updates.

---

# Reporting a Vulnerability

If you discover a security vulnerability, please **do not** create a public GitHub issue.

Instead, report it privately by contacting the project maintainer.

Please include the following information:

- API Shield version
- Node.js version
- Operating System
- Steps to reproduce
- Proof of concept (if available)
- Potential impact
- Suggested fix (optional)

---

# Response Timeline

We aim to respond according to the following timeline:

| Stage | Expected Time |
|--------|---------------|
| Initial Response | Within 48 hours |
| Vulnerability Assessment | Within 5 business days |
| Security Patch | As soon as possible |

Response times may vary depending on the complexity and severity of the issue.

---

# Responsible Disclosure

Please follow responsible disclosure practices:

- Do not publicly disclose the vulnerability before a fix is available.
- Allow maintainers reasonable time to investigate and resolve the issue.
- Avoid accessing, modifying, or deleting data that does not belong to you.
- Do not perform actions that could disrupt users or services.

---

# Security Best Practices

API Shield is designed to improve API security, but developers should also follow general security best practices, including:

- Keep dependencies up to date.
- Store secrets securely using environment variables.
- Use HTTPS in production.
- Validate all incoming user input.
- Apply the principle of least privilege.
- Monitor application logs regularly.

---

# Third-Party Dependencies

API Shield depends on trusted open-source packages.

Security updates for dependencies are applied regularly to reduce potential risks.

---
